## 4. USE CASE DEMO STRUCTURE (repeated pattern across all 4)

Ian used an identical 6-part structure for every use case demo:

**Step 1 — Name the use case and its strategic context**  
Opens with "why this matters" framed around a specific failure sellers have (not just the tactic). Every use case name ties to a strategic skill, not a feature.

**Step 2 — Show the prompt verbatim (screen share)**  
Reads the prompt aloud while it's on screen. Always specifies what fields the user must customize (their name, company, product, target account). Never hides the prompt mechanics.

**Step 3 — Live run with a real (or real-ish) example**  
Uses Workiva + KeyBank throughout (randomly selected from Fortune 500/Fortune top 40 tech companies). For email prompts, uses accounts he personally managed at Salesforce. Runs the full output — doesn't cherry-pick.

**Step 4 — Narrate what's impressive in the output**  
Reads specific highlights: "Look at this — it mapped their top five goals and showed exactly how Workiva helps with each one." Points to the "Sources" button in ChatGPT Plus and encourages verifying the citations.

**Step 5 — Iteration moment**  
Shows at least one example of re-prompting to make output sound more personal/human. "I didn't love how I said X, so I prompted it and asked it to make the first sentence sound more like me." Reinforces: it's a thinking partner, not a copy machine.

**Step 6 — Audience engagement + transition**  
Asks for a reaction in chat ("give me a little applause," "drop a fire"), reads results, then bridges to the next use case with one-sentence framing.
