## Operating Principles (How to Run the Business)

- Work in focused conversations within the right project. Do not maintain one endless thread.
- Capture conversation outputs into knowledge files.
- Build playbooks with every engagement, not just deliverables.
- Price based on value, not time.
- Treat early clients as mutual investments.
- Be honest about capability boundaries.

**The Client Acquisition Pattern:** Live event or workshop creates room credibility → one person reaches out with a specific problem → 60-minute discovery call and scope document within 24 hours → Phase 1 delivered in one week → client is delighted → client tells everyone → referrals follow.

**Build vs Run:** every engagement splits into what must be *built* (an agent you watch and steer: Cowork or Claude Code) and what must *run* it (the delivered Project on the client's connector, optionally an always-on operator: OpenClaw or Make). Ask "what has to be built, and what has to run it."

**The Auth Boundary:** the agent has the hands and the speed; the human holds the keys. Read-only-first on any live client system.

**The Blueprint Is the Asset:** encode domain knowledge into a reusable build-skill once; any agent then runs it from a one-line prompt. The tool is interchangeable; the skill is the product. Cowork forges the skill, OpenClaw scales it.

## Client Onboarding (Two Patterns)

**1:1 Coaching Acquisition Pattern:** UYSP markets the offering to all tiers → member books through UYSP checkout and Calendly → Nick's intake auto-sends the AI Audit Prompt + Intake Form → Nick runs LinkedIn analysis via Claude Chrome → the coaching-prep skill generates the session brief → Nick delivers, builds the project, follows up → 7-day and 30-day check-ins generate referrals and rebooks. (Full lifecycle in methodologies-coaching-lifecycle.md.)

**External Implementation Onboarding:** discovery and scoping (who, exact manual workflow, emotional pain point / "30-day win" in their words, tech stack, scale, voice samples; phase and price the engagement) → confirm live schema from a real reference file → build context layer → build skills one at a time → populate client folders (read-only inventory before bulk creation) → write CLAUDE.md → pre-delivery audit → deliver live on a call → enrich and iterate post-delivery. (Full sequence in methodologies-build-chain.md.)
