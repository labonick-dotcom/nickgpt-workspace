**How often it'll happen**

More than it should. Google OAuth refresh tokens expire when:
- The connected app is still in "testing" mode vs. published to production -- testing mode tokens die every 7 days, this is the most common culprit
- The account hasn't used the connection in 6+ months
- Google flags something unusual and revokes access
- Claude's side has an outage or token refresh failure

If the Drive connector was set up through Claude's native connection and it's expiring frequently, the likely cause is the OAuth app sitting in testing mode. That's a developer-side fix -- whoever owns the Google Cloud project the OAuth credentials belong to needs to publish it to production. After that, tokens last until the user revokes them.
